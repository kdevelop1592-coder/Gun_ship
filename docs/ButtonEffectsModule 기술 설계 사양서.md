### ButtonEffectsModule 기술 설계 사양서

#### 1\. 모듈 개요 및 설계 철학 (Module Overview & Design Philosophy)

현대적인 게임 개발 아키텍처에서 UI 애니메이션은 단순한 시각적 장식을 넘어, 사용자 피드백의 직관성을 결정짓는 핵심 인터페이스 요소입니다. 본 ButtonEffectsModule은 Godot 4.x 환경에서 UI 애니메이션의 모듈화를 통해 개발 생산성을 극대화하고 프로젝트 전반의 시각적 일관성을 확보하도록 설계되었습니다.본 모듈은 개별 버튼마다 중복된 코드를 작성하는 전통적인 방식 대신, \*\*'컴포넌트 기반 아키텍처(Component-based Architecture)'\*\*를 채택합니다. 이는 애니메이션 로직을 독립적인 노드 단위로 캡슐화하여, 어떤 버튼 노드에도 자식으로 부착하기만 하면 즉각적인 상호작용 기능을 부여할 수 있음을 의미합니다.

##### 구조적 설계 원칙

* **전역 접근성 및 확장성:**  Node2D를 상속하며 class\_name을 정의하여 프로젝트 어디에서든 인스펙터를 통해 쉽게 추가할 수 있습니다.  
* **구조적 전제 조건:**  런타임 최적화와 코드 간결성을 위해 본 모듈은 항상 버튼 노드의 '직속 자식(Direct Child)'으로 배치되어야 합니다. 이는 불리언(Boolean) 체크나 복잡한 노드 탐색 없이 get\_parent()만으로 안정적인 참조를 보장합니다.  
* **엔진 최적화:**  Godot 4.x의 최신 기능인 pivot\_offset\_ratio와 Tween 시스템을 적극 활용하여 유연하고 가벼운 시스템을 지향합니다.

#### 2\. 구성 변수 및 익스포트 파라미터 (Configuration Variables & Export Parameters)

본 모듈은 디자이너와 엔지니어가 코드 수정 없이 인스펙터 창에서 애니메이션의 물리적 거동을 미세 조정할 수 있도록 파라미터화된 인터페이스를 제공합니다.

##### 핵심 익스포트 변수 리스트

1. **ease\_type**  **(Tween.EaseType):**  애니메이션의 가감속 곡선을 결정합니다. (기본값: Tween.EASE\_OUT)  
2. **trans\_type**  **(Tween.TransitionType):**  애니메이션의 전환 스타일을 정의합니다. (기본값: Tween.TRANS\_SINE)  
3. **anim\_duration**  **(float):**  애니메이션의 총 재생 시간입니다.  **0.07초** 는 입력에 대한 즉각적인 반응성과 시각적 부드러움을 동시에 충족하는 최적의 임계값입니다.  
4. **scale\_amount**  **(Vector2):**  상호작용 시 목표 크기입니다. 기본값 \*\*(1.1, 1.1)\*\*은 명확한 시각적 깊이감을 제공합니다.  
5. **rotation\_amount**  **(float):**  호버 시 적용될 회전 각도의 절댓값입니다.  **3.0**  내외의 수치는 정적인 UI에 생동감을 부여합니다.

#### 3\. 초기화 로직 및 신호 연결 (Initialization & Signal Connectivity)

모듈의 \_ready() 함수는 런타임 안정성과 연출의 정확성을 확보하기 위해 다음과 같은 시스템 설정 프로세스를 수행합니다.

##### 초기화 및 방어적 설계

1. **부모 노드 참조:**  get\_parent() as Button을 통해 제어 대상을 획득합니다. 명시적 타입 캐스팅을 통해 런타임 에러를 방지합니다.  
2. **피벗(Pivot) 최적화:**  button.pivot\_offset\_ratio \= Vector2(0.5, 0.5)를 설정합니다. 이는 Godot 4.x의 핵심 기능으로, 버튼의 크기와 상관없이 중앙을 기준으로 애니메이션이 발생하게 합니다. 픽셀 기반 오프셋 계산이 필요 없는  **'크기 무관형(Size-agnostic)'**  설계를 가능케 합니다.  
3. **상태 기반 신호 바인딩:**  mouse\_entered와 mouse\_exited 신호를 하나의 함수(on\_mouse\_hovered)에 연결하되, bind(true/false)를 활용하여 페이로드를 전달합니다. 이는 내부 변수 체크가 필요 없는 '무상태(Stateless)' 로직 구현을 가능하게 합니다.  
4. **방어적 클릭 처리:**  pressed 신호를 사용하여 클릭 애니메이션을 처리합니다. 이는 button\_down/up 신호 사용 시 발생할 수 있는 '버튼 밖으로 드래그하여 떼는 경우'의 애니메이션 꼬임 현상을 방지하는 전략적 선택입니다.

#### 4\. 트윈(Tween) 시스템 관리 및 리셋 메커니즘 (Tween Management & Reset Mechanism)

상태 변화가 빈번한 UI 환경에서는 이전 트윈과 새로운 트윈 간의 충돌을 방지하는 것이 자원 관리의 핵심입니다.

##### reset\_tween() 실행 시퀀스

* **리소스 보호 (**  **kill**  **):**  기존 트윈 인스턴스가 존재할 경우 kill()을 호출하여 즉시 중단합니다. 이는 애니메이션의 중첩 현상을 방지하고 메모리 누수를 차단합니다.  
* **객체 초기화 (**  **create\_tween**  **):**  새 트윈을 생성하고 인스펙터에서 설정한 ease\_type과 trans\_type을 체이닝 방식으로 즉시 주입합니다.  
* **병렬 제어 활성화 (**  **set\_parallel**  **):**  set\_parallel(true)를 호출하여 크기(Scale)와 회전(Rotation) 변이가 독립적이면서도 동시에 발생하도록 설정합니다. 이는 복합적인 시각 연출을 위한 기술적 필수 단계입니다.

#### 5\. 상태별 상호작용 구현 (State-Specific Interaction Implementation)

##### 5.1 마우스 호버 (on\_mouse\_hovered)

삼항 연산자를 활용하여 간결하면서도 강력한 상태 전환 로직을 구현합니다.

* **크기 전환:**  hovered 상태에 따라 scale\_amount 또는 Vector2(1, 1)로 트윈합니다.  
* **무작위 회전 구현:**  고정된 연출을 탈피하기 위해 rotation\_amount \* \-1, 1.pick\_random() 공식을 사용합니다. 사용자가 마우스를 올릴 때마다 무작위로 왼쪽 또는 오른쪽으로 회전하여 동적인 UX를 제공합니다.

##### 5.2 버튼 클릭 (on\_button\_pressed)

클릭 시에는 호버링 중인 상태를 즉시 무효화하고 명확한 피드백을 주어야 합니다.

* **애니메이션 오버라이딩:**  트윈 호출 시 .from() 메소드를 사용합니다. 이는 현재 상태와 상관없이 애니메이션의 시작점을 강제로 지정함으로써, 클릭 피드백이 항상 일정한 강도로 "톡 쏘는(Snappy)" 느낌을 주도록 보장합니다.

#### 6\. 결론 및 확장 제언 (Conclusion & Extension Strategy)

본 ButtonEffectsModule은 Godot 4.x의 컴포넌트 시스템과 고급 트윈 제어 기법을 결합하여, 유지보수가 용이하고 확장이 자유로운 UI 시스템의 표준을 제시합니다.

##### 시스템 확장성

본 설계는 시각 효과에 국한되지 않는 유연한 구조를 가지고 있습니다. 향후 상호작용 지점에  **사운드 효과(Sound Effects)**  재생 로직을 통합하거나, 파티클 시스템을 연동함으로써 시각과 청각이 어우러진 통합형 UI 피드백 시스템으로 쉽게 확장할 수 있습니다.**핵심 가치 요약:**  본 모듈은 최소한의 구현 비용으로 게임 전체의 폴리싱 품질을 극대화하며, 복잡한 로직 없이도 고품질의 상호작용 경험을 보편화하는 강력한 개발 자산을 제공합니다.
