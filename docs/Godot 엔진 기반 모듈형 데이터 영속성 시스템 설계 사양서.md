### Godot 엔진 기반 모듈형 데이터 영속성 시스템 설계 사양서

#### 1\. 시스템 아키텍처 개요 및 설계 철학

현대적인 게임 엔진 아키텍처에서 데이터 영속성(Persistence) 시스템의 성패는 '결합도(Coupling)의 최소화'에 달려 있습니다. 본 설계서는 Godot 엔진의 \*\*커스텀 리소스(Custom Resource)\*\*와  **노드 그룹핑**  메커니즘을 결합하여, 특정 노드나 속성에 종속되지 않는 고도로 모듈화된 저장 시스템을 제안합니다.이 아키텍처의 핵심 이점은 유연성입니다. 기존의 하드코딩된 방식은 새로운 데이터 필드가 추가될 때마다 직렬화 로직을 수동으로 수정해야 했으나, 본 시스템은 리소스를 범용적인 데이터 컨테이너로 활용합니다. 개발자는 코드 한 줄 수정 없이 에디터 상에서 저장할 속성을 동적으로 정의할 수 있으며, 이는 프로젝트 규모 확장에 따른 유지보수 비용을 획기적으로 절감합니다.이하에서는 데이터를 담는 추상화 계층인 리소스 설계부터 구체적인 파일 입출력 전략까지 기술합니다.

#### 2\. 데이터 컨테이너 설계: MapSaveData 커스텀 리소스

데이터 무결성을 유지하기 위해서는 직렬화된 데이터를 안전하게 보관할 추상화 계층이 필수적입니다. 이를 위해 Resource를 상속받는 전용 컨테이너를 설계합니다.

##### 2.1 구조 설계 및 상속

MapSaveData 클래스는 class\_name을 명시하여 엔진 내에서 고유 타입으로 식별되어야 합니다.

* **기본 클래스:**  Resource 상속 (엔진 고유의 직렬화/역직렬화 기능 활용).  
* **핵심 변수:**  @export var data: Dictionary \= {}  
* 전역 저장소 역할을 수행하며, 노드 경로를 키로, 해당 노드의 상태 정보를 값으로 가집니다.  
* @export 어노테이션은 ResourceSaver가 이 딕셔너리를 파일로 기록할 수 있도록 보장합니다.

##### 2.2 데이터 매핑 논리

데이터는 Map 노드에서 실행되는 get\_path\_to(saver\_node) 메서드를 통해 생성된 \*\*상대 경로(Node Path)\*\*를 키(Key)로 사용합니다.

* **Key:**  각 객체의 고유한 경로 정보를 활용하여 맵 내 데이터 중복을 방지합니다.  
* **Value:**  SaverNode가 추출한 부모 노드의 속성 명칭과 값의 쌍(Dictionary)입니다.이러한 구조는 데이터와 실제 노드 간의 느슨한 결합을 구현하여, 씬 구조가 변경되더라도 경로 기반의 정합성을 유지하게 합니다.

#### 3\. 데이터 캡처 및 복원 메커니즘: SaverNode 구현

개별 객체의 데이터 독립성은 전체 시스템의 확장성을 결정합니다. 이를 위해 부모 노드의 데이터를 캡처하고 복원하는 독립 컴포넌트인 SaverNode를 구현합니다.

##### 3.1 핵심 기능 및 그룹 관리

* **그룹 상수화:**  매직 스트링 오염을 방지하기 위해 const saver\_node\_group \= "saver\_node"를 정의하여 노드 관리를 표준화합니다.  
* **get\_save\_dictionary()**  **:**  get\_parent()를 통해 부모 객체에 접근하고, 사용자 정의 속성 배열인 properties\_to\_save를 순회합니다. parent.get(prop) 메서드를 사용하여 동적으로 값을 추출하고 딕셔너리로 반환합니다.  
* **apply\_save\_dictionary(node\_data)**  **:**  로드된 데이터를 parent.set(prop, value)를 통해 부모 노드의 속성에 리플렉션 방식으로 주입합니다.

##### 3.2 동적 속성 접근 및 범용성

if prop in parent 체크 로직을 통해 해당 속성의 존재 여부를 사전에 검증합니다. 이러한 방식은 SaverNode가 특정 노드 타입(CharacterBody, Sprite 등)에 구속되지 않고, 어떤 노드 아래에 배치되더라도 해당 노드의 공개된 속성에 즉시 접근할 수 있는 강력한 다형성을 제공합니다.

#### 4\. 에디터 워크플로우 최적화: @tool 및 프로퍼티 유효성 검사

개발자의 실수(오타 등)를 원천 차단하고 생산성을 높이기 위해 @tool 스크립트와 에디터 인터페이스 자동화를 도입합니다.

##### 4.1 "Black Magic" 자동화 구현

* **에디터 힌트:**  \_ready 함수 내에서 if Engine.is\_editor\_hint(): update\_property\_list()를 호출하여 씬을 여는 즉시 자동화 로직이 활성화되도록 합니다.  
* **속성 리스트 오버라이드:**  \_validate\_property 함수를 통해 properties\_to\_save 배열의 인스펙터 동작을 수정합니다.  
* **데이터 포맷:**  property.hint\_string에 TYPE\_STRING / PROPERTY\_HINT\_ENUM : options 형태의 특수 포맷을 적용하여, 일반 텍스트 입력창을 부모 노드의 속성 리스트가 담긴  **드롭다운 메뉴** 로 강제 전환합니다.  
* **상태 갱신:**  notify\_property\_list\_changed()를 호출하여 인스펙터 UI를 실시간으로 새로고침합니다.

##### 4.2 전략적 가치 분석

수동 문자열 입력 방식은 오타에 취약하고 유지보수가 어렵지만, 제안된 드롭다운 방식은 부모 노드의 get\_property\_list()를 기반으로 유효한 속성만 노출하므로 데이터 정합성을 에디터 단에서 강제할 수 있습니다.

#### 5\. 맵 관리 및 전역 접근 아키텍처

대규모 프로젝트에서 여러 구역의 데이터를 혼선 없이 관리하기 위해 맵 기반 ID 체계와 싱글톤 패턴을 결합합니다.

##### 5.1 고유 ID 및 싱글톤 설계

* **Enum ID 시스템:**  Map 클래스 내에 Enum을 정의하고, 각 항목에  **수동으로 고유 정수 값을 할당** 합니다 (예: MAP\_FOREST \= 1). 이는 엔진의 자동 할당에 의한 데이터 덮어쓰기나 버전 간 충돌을 방지하기 위함입니다.  
* **전역 접근:**  static var current\_map: Map 변수를 선언하고, \_ready 시점에 current\_map \= self로 초기화하여 외부 시스템(UI, 전투 시스템 등)이 표준 프로콜을 통해 저장 기능을 호출할 수 있게 합니다.

##### 5.2 통합 저장 로직

전역 맵 스크립트는 get\_tree().get\_nodes\_in\_group(saver\_node\_group)을 통해 씬 내의 모든 SaverNode를 수집합니다. 이후 get\_path\_to(saver)를 통해 맵 리소스를 구성하고 파일을 영속화합니다.

#### 6\. 데이터 영속화 및 파일 시스템 전략

최종적으로 데이터는 사용자 로컬 환경에 물리적 파일로 안전하게 기록되어야 합니다.

##### 6.1 파일 시스템 전략

* **경로 설정:**  배포 후 수정이 가능하도록 user:// 경로를 사용하며, 안정성을 위해 DirAccess.make\_dir\_absolute()를 통해 저장 디렉토리 존재 여부를 상시 확인합니다.  
* **파일명 규칙:**  prefix\_ \+ map\_id \+ .extension (예: map\_save\_1.tres) 형식을 준수하여 각 맵 데이터의 고유성을 보장합니다.  
* **확장자 선택:**  
* **.tres**  **(Text Resource):**  개발 단계에서 사용하며, Git 협업 시 변경 사항 추적과 디버깅 가독성이 우수합니다.  
* **.res**  **(Binary Resource):**  최종 배포 단계에서 사용하며, 일반 사용자의 임의 수정을 방지하고 보안성을 높입니다.

##### 6.2 보안 고려사항 및 결론

커스텀 리소스 로딩 방식은 코드 실행의 잠재적 위험이 있으나, 이는 주로 외부 소스(Steam Workshop 등)의 파일을 공유할 때의 문제입니다. 로컬 저장 파일만을 관리하는 단일 플레이어 게임 환경에서는 시스템 효율성과 모듈성 측면에서 가장 합리적인 선택입니다.본 설계는 Godot 엔진의 강점을 극대화하여 개발 편의성과 시스템 안정성을 동시에 확보한 현대적인 영속성 솔루션을 제공합니다.
